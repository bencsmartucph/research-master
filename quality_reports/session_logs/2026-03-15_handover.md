## Critical Technical Notes (from MEMORY.md `[LEARN]` tags)

These must be followed without exception in any data or code work:

1. **pyreadstat is mandatory** for `.dta` files — not `pandas.read_stata`. Preserves value labels, which are critical for ISSP ZA files.

2. **Multi-encoding fallback for ESS CSVs** — try `utf-8` → `latin-1` → `cp1252`. The Im ESS file (401.6MB) requires `latin-1`.

3. **ISCO-08 must be truncated before merging to task scores** — ESS stores 4-digit codes; `isco08_3d-task3.csv` uses 3-digit. Use `df['isco08'].astype(int) // 10`. Failing to truncate produces ~0% match rate with no error.

4. **`posit_income_change` is pre-constructed** in `cicollini_2025/essprt-all.dta`. Do not reconstruct from ESS income variables.

5. **ESS is NOT panel data** — repeated cross-sections. Do not treat as individual panel. Exception: rotating panel component in some waves.

6. **Code style:** Sequential inline Python, no function definitions, section headers (`# --- Config ---`, `# --- Load ---`, etc.), inline `assert` for validation.

7. **Baccini path discipline** — use `Data/individualdata.dta` and `Data/districtdata.dta`. Do NOT use `Raw Data/` components directly.

8. **Euroscepticism data** — `.dta` and `.csv` are the same dataset; use `.dta`.

9. **ISSP ZA-files** — always load `meta.variable_value_labels` before recoding; never assume code meanings.
